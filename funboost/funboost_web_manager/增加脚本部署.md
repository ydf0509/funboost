funboost web manager 增加一个脚本部署 的功能模块

# 1. 只负责本机部署即可，不考虑跨机器远程部署

# 2. funboost web manager 左侧增加一个 “脚本部署” 的导航栏

可以部署任何脚本，和脚本是不是funboost消费程序没关系
要求：

## 2.1 页面中设置项目根目录，

## 2.2 页面中有git pull按钮，点击git pull可以显示完整的git命令返回，
  如果git pull拉取失败或冲突等，需要提醒用户，免得用户继续部署

## 2.3 页面中可以设置环境变量

## 2.4 页面中设置用户需要运行的命令
用户点击重启按钮，可以重启用户需要运行的命令

## 2.5 需要读取 nb_log_config.py 中的 LOG_PATH配置
 from nb_log import nb_log_config_default 就能自动读取了。
 SYS_STD_FILE_NAME = os.environ.get("SYS_STD_FILE_NAME")

所以你在运行脚本前要设置这个 SYS_STD_FILE_NAME 环境变量独一无二，并保存到redis，以供后续查看
SYS_STD_FILE_NAME建议是 脚本名字 + 启动时间字符串 + uuid 拼接起来，

## 2.6 运行部署脚本需要兼容win和linux

## 2.7 这些页面中填写的信息，保存到redis
from funboost import RedisMixin 就可以，用里面redis

## 2.8 脚本部署中，要支持查看滚动日志，支持按关键字搜索日志
就是查看日志默认进去日志滚动
但要支持搜索关键字，并高亮显示关键字，支持按时间范围搜索日志。

搜索日志，要搜索的文件名 不是精确的 SYS_STD_FILE_NAME ，因为nb_log会自动切割日志 生成的文件名字带有其他前缀， 所以文件名只要包含 SYS_STD_FILE_NAME 字符串即可认为是这个脚本的日志。


## 2.20 需要分别保存每台机器ip对应的以上这些信息，不要混。
因为不同的机器ip，可能部署不同的脚本，以及项目根目录不一样。
