# Funboost Web Manager 新增模块：脚本部署

## 目标
在 Funboost Web Manager 左侧导航栏增加“脚本部署”功能模块，用于管理本机上的任意脚本的部署、启动、重启和日志查看。

## 核心要求
1. **本机部署**：只管理当前 Web 服务所在机器上的部署，不涉及远程机器。
2. **区分多机器**：不同机器的 Web Manager 实例可能共用同一个 Redis，因此保存到 Redis 的配置信息需要以机器 IP 作为区分标识（例如：`script_deploy:{ip}:{deploy_name}`）。


## 页面结构
- 左侧导航栏新增“脚本部署”链接。
- 点击后进入部署列表页，展示所有已配置的部署（部署名称、状态、操作）。
- 点击某个部署进入详情页，可查看/编辑配置、控制进程、查看日志。

## 详细功能点

### 1. 部署配置（每个部署独立保存）
- **部署名称**：唯一标识（用户输入）。
- **项目根目录**：脚本所在的本地绝对路径。
- **环境变量**：键值对形式，支持增删改。
- **启动命令**：例如 `python main.py`，点击“启动”时会执行该命令。
- **Git 操作**：提供一个“Git Pull”按钮，点击后执行 `git -C {项目根目录} pull`，并将命令输出实时显示在界面上。若拉取失败（如冲突），需用红色高亮提示，并禁用启动按钮直到问题解决。
- **保存**：以上配置保存到 Redis，键格式：`script_deploy:{本机IP}:{部署名称}:config`。

### 2. 进程管理
- 每个部署应能独立启动、停止、重启。
- 启动时记录子进程的 PID 到 Redis（键：`script_deploy:{IP}:{部署名称}:pid`），同时记录启动时间。
- 停止时通过 PID kill 进程（使用 `os.kill(pid, signal.SIGTERM)`，兼容 Windows 需使用 `taskkill` 或 `terminate`）。
- 重启操作：先停止再启动。
- 页面需实时显示当前进程状态（运行中/已停止）和启动时间。
- 启动脚本时候，你可以无脑加个命令行参数，例如 --deploy_flag=一个uuid ，之后你可以搜索这个uuid来找到这个脚本是否存活。 例如win机器可能每天关机，第二天打开网页，要知道这个脚本是否是在运行的。

### 3. 日志处理
- **日志重定向**：在启动命令前，设置环境变量 `SYS_STD_FILE_NAME` 为唯一标识，例如 `{部署名称}_{时间戳}_{uuid}`。该环境变量会被 `nb_log` 库读取，用于生成日志文件名。
- **日志存储**：`nb_log` 默认日志路径由 `nb_log_config.py` 中的 `LOG_PATH` 决定，可从 `nb_log.nb_log_config_default.LOG_PATH` 获取。日志文件名包含 `SYS_STD_FILE_NAME`。
- **日志查看**：
  - 在部署详情页提供日志查看区域，默认显示最近 1000 行日志，并支持“自动刷新”（每 3 秒拉取最新内容）。
  - **关键字搜索**：用户输入关键字后，前端实时高亮所有匹配行。
  - **时间范围过滤**：用户可选择时间范围（如“最近1小时”、“最近1天”），后端根据日志文件的修改时间或文件名中的时间戳过滤，返回匹配时间段内的日志。
- **日志文件定位**：需要扫描 `LOG_PATH` 目录下所有文件名包含 `SYS_STD_FILE_NAME` 的文件，并按修改时间合并排序，确保日志顺序正确。

### 4. 兼容性
- 所有操作需兼容 Windows 和 Linux（包括路径处理、进程终止等）。

### 5. 数据持久化
- 使用 `funboost.utils.redis_manager.RedisMixin` 操作 Redis，确保连接复用。
- 所有配置、PID 等信息均保存在 Redis 中，Web 服务重启后能恢复状态。

## UI 设计提示
- 采用 Bootstrap 或类似组件库，保持与现有 Web Manager 风格一致。
- 部署列表页使用表格，包含“部署名称”、“状态”、“操作”（查看详情、删除）。
- 部署详情页分为几个卡片区域：
  - 基本信息（部署名称、项目根目录）
  - 环境变量（键值对表格，可增删行）
  - 启动命令（文本输入框）
  - Git 操作（按钮 + 输出框）
  - 进程控制（启动、停止、重启按钮 + 状态显示）
  - 日志查看（带搜索框、时间范围选择、日志滚动区域）

## 技术实现要点
- 后端使用 Flask 路由（如 `/deploy/list`, `/deploy/<name>`, `/deploy/<name>/start` 等）。
- 进程管理使用 `subprocess.Popen`，注意捕获 stdout/stderr 以便后续记录到日志文件。
- 日志查看接口需支持参数：`deploy_name`, `keyword`, `time_range`，返回日志内容列表。
- 为简化实现，日志文件读取可采用按行读取并缓存的方式。

## 提示
- 利用 `funboost.utils` 中的工具类，如 `RedisMixin`、`get_logger` 等。
- 参考现有 Web Manager 代码结构（`funboost/funboost_web_manager/app.py` 和 `functions.py`）。
- 代码注释要清晰，特别是跨平台兼容部分。

## 最终交付物
- 新增的 HTML 模板文件（放在 `funboost_web_manager/templates/` 下）。
- 新增的路由和业务逻辑代码（放在 新建 `deploy.py` 模块并导入）。
- 确保与现有登录认证集成（需要 `@login_required`）。
